API Documentation

User Management and Subscriptions

Before Match ID

Introduction

This document outlines the Application Programming Interface (API) endpoints for user authentication, profile management, and subscription handling. It details the request formats, expected responses, and the state changes associated with different user subscription statuses within the application.

Base URL: http://localhost:5000

Chapter 1: User Onboarding and Profile Management (Before Any Subscription)

This chapter covers the initial steps a user takes, including requesting and verifying OTP (One-Time Password) for authentication, and subsequently updating their profile information.

1.1 OTP Request

POST /auth/request-otp

Sends an OTP to the provided phone number. This is the first step in user registration or login.

Request Body Example:

{
  "phone": "+917004653005"
}

Request Body Fields:

Field Type Required Description
phone string Yes The phone number to which the OTP will be sent, including the country code (e.g., "+91").

Successful Response (200 OK):

{
  "success": true,
  "data": null,
  "error": [],
  "message": "OTP sent successfully"
}

Response Fields:

Field Type Description
success boolean Indicates if the OTP request was successful.
data null No specific data is returned for this endpoint on success.
error array An empty array if no errors occurred.
message string A descriptive message indicating the outcome.

1.3 Update User Profile

Authentication: Bearer Token

Request Body Example:

  "nickname": "Gullu",
  "position": "bottom",

  "dob": "1998-07-01",
  "location": {
    "lat": 25.645,
    "lng": 85.143

Chapter 2: Basic Plan Subscription

This chapter details the process of subscribing to the "Basic" plan and the subsequent changes in user capabilities and API responses.

2.1 Initiate Basic Plan Subscription

POST /subscription

Activates a subscription for the authenticated user based on the provided plan.

Authentication: Bearer Token

Request Body Example:

{ "plan": "basic" }

Request Body Fields:

Field Type Required Description
plan string Yes The name of the plan to subscribe to (e.g., "basic").

Successful Response (200 OK):

{
  "success": true,
  "data": {
    "subscription": {
      "_id": "686d1ce2c1e583150b5faaab",
      "userId": "686d1bde552bb16614d7927a",
      "__v": 0,
      "endDate": "2025-08-07T13:28:02.066Z",
      "isActive": true,
      "plan": "basic",
      "startDate": "2025-07-08T13:28:02.066Z"
    },
    "isPremium": true,
    "subscriptionEndsOn": "2025-08-07T13:28:02.066Z",
    "superMessagesLeft": 0,
    "swipes": 30,
    "monthlyTokenAllowance": 0
  },
  "error": [],
  "message": "Subscription activated"
}

Response Fields:

Field Type Description
success boolean Indicates if the subscription activation was successful.
data object Contains details about the newly activated subscription and updated user status.
subscription object Details of the subscription record.
_id string Unique subscription identifier.
userId string ID of the user who subscribed.
endDate string Date when the subscription will end.
isActive boolean Always true for an active subscription.
plan string The subscribed plan name.
startDate string Date when the subscription started.
isPremium boolean true after activating a premium plan.
subscriptionEndsOn string The calculated end date of the subscription.
superMessagesLeft number Number of super messages granted by the plan.
swipes number Daily swipe limit granted by the plan.
monthlyTokenAllowance number Monthly allowance for tokens (e.g., for super messages).
error array An empty array if no errors occurred.
message string A descriptive message indicating the outcome.

Authentication: Bearer Token
